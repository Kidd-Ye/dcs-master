# DCS Intent Routing

当用户开口很短、很散、信息不完整时，先用本文件判断：

1. 这是哪一种意图
2. 应该进入哪一种模式
3. 先加载哪份资料
4. 第一轮该怎么回

## 术语纠正

- 用户如果写了 `开发商表示`，优先按 `开发商标识` 理解。
- 用户如果说“看不到 ISV 项目类型”，不要直接等同于“选不出开发商标识”。

## 操作类问法的额外规则

- 只要用户是在问具体平台操作，除了正常意图路由，还要额外加载 `rich-answering.md` 和对应主题的详细知识文档。
- 第一轮输出不能退化成纯文字说明；如果知识库里已有截图与来源链接，默认给出 1 到 3 张相关图和至少 1 个原文链接。
- 快速模式也适用这条规则，只是把图片压缩到 1 张关键图或保留 1 个关键来源链接。

## 一级意图分类

### A. 从零学习 / 带练

常见原话：

- `带我一步步创建 DCS 项目`
- `我完全不会 DCS，从零教我`
- `像教练一样带我做`
- `第一次用协同开发平台`
- `为什么新建项目只有定制开发`
- `为什么看不到 ISV标准产品开发`

进入模式：

- 教练模式

优先加载：

1. `coach-playbooks.md`
2. `response-templates.md`
3. `milestone-checklists.md`
4. `rich-answering.md`（如果用户是在问具体操作）
5. 再按阶段补 `getting-started.md` 或其他资料

第一轮输出：

- 用“第一次开场”模板
- 不要一上来讲全流程细节
- 如果用户问的是具体操作怎么做，优先切成图文版回答，并附原文链接
- 如果用户的问题是“为什么没有 ISV 项目类型”，优先按项目类型授权问题分流，而不是直接按开发商标识问题分流
- 如果当前目标是登录、创建项目、关联、构建、发布这类浏览器可执行操作，必须在首轮追加一句可选入口：
  - `如果你想我直接打开浏览器代操作，也可以直接回：帮我打开浏览器<当前任务>`

## B. 已有项目，继续往下做

常见原话：

- `项目已经建好了，下一步怎么推元数据`
- `仓库已经有了，怎么构建`
- `我现在要发布到天梯`

进入模式：

- 教练模式

优先加载：

1. `coach-playbooks.md`
2. `milestone-checklists.md`
3. `rich-answering.md`（如果用户是在问具体操作）
4. 对应阶段资料

第一轮输出：

- 先确认用户当前阶段的完成定义是否满足
- 再推进下一个里程碑
- 如果问题本身是“下一步怎么做 / 这个操作怎么点”，优先给图文版最小动作

## C. 快速办事

常见原话：

- `别解释，直接告诉我怎么做`
- `给我最短步骤`
- `我赶时间，直接说操作`

进入模式：

- 教练模式的快速分支

优先加载：

1. `response-templates.md`
2. `rich-answering.md`
3. 对应操作资料

第一轮输出：

- 用“快速模式”模板
- 保留关键风险提醒，但去掉大段背景解释
- 若知识库已有图文，至少附 1 张关键图或 1 个关键原文链接

## D. 直接浏览器代操作 / 页面协作

常见原话：

- `帮我打开浏览器登录 DCS`
- `你直接帮我创建项目`
- `帮我在页面里点`
- `帮我看当前页面下一步怎么操作`

进入模式：

- 浏览器协作模式

优先加载：

1. `response-templates.md`
2. `browser-automation.md`
3. `rich-answering.md`（如果用户还要求带图或讲解）
4. 对应阶段资料

第一轮输出：

- 如果用户已经明确授权，直接进入浏览器协作模式
- 先确认当前目标和当前页面
- 每轮只推进 1 到 3 个动作
- 密码、删除、发布部署、生产环境相关动作必须再次确认

## E. 构建失败 / 发布失败 / 部署失败

常见原话：

- `DCS 构建失败了`
- `构建一直没显示成功或者失败`
- `发布时报未查询到关联环境`
- `部署成功了但页面没出来`
- `在线构建报依赖错误`

进入模式：

- 诊断模式

优先加载：

1. `intent-routing.md`
2. `troubleshooting.md`
3. 必要时 `response-templates.md`

第一轮输出：

- 先复述错误
- 用“排障输出”模板
- 如果证据不足，只追一个最小证据
- 如果是“构建一直没有结果”，优先追 `构建日志`
- 若构建日志也无法提示问题，明确引导用户联系 DCS 云之家沟通群中的林心梅或谭鼎鹏老师

## F. 质量扫描不过 / 要修漏洞

常见原话：

- `SonarQube 不通过`
- `Fortify 有高危怎么处理`
- `帮我看质量报告`
- `你直接帮我修漏洞`
- `构建成功了，但是质量结论一直没出来`

进入模式：

- 诊断模式或修复模式

判断规则：

- 只有报告，没有代码：先诊断
- 有仓库并要求改代码：直接修复模式

优先加载：

1. `release-and-quality.md`
2. `troubleshooting.md`
3. `response-templates.md`
4. 如需修复，再看真实仓库

第一轮输出：

- 先确认是 Fortify 还是 SonarQube
- 再确认是否有文件、行号、规则名
- 如果是“质量结论一直没出来”，先分流：
  - 构建内容很多，扫描耗时较长
  - 扫描引擎异常，导致整体质量结论无法输出
- 若怀疑扫描引擎异常，明确引导用户联系 DCS 运营人员

## G. 插件开发 / Gradle / IDEA 小助手

常见原话：

- `我要做插件开发`
- `怎么初始化 DCS 的 Gradle 工程`
- `IDEA 小助手怎么用`
- `DCS 构建和本地 Gradle 不一致`

进入模式：

- 教练模式或修复模式

判断规则：

- 要搭环境、学流程：教练模式
- 已经有工程且构建坏了：修复模式

优先加载：

1. `build-and-repo.md`
2. `coach-playbooks.md`
3. `troubleshooting.md`

## H. 概念解释

常见原话：

- `轻扩展和定制开发有什么区别`
- `云、应用、页面是什么关系`
- `什么情况下用 DCS`

进入模式：

- 教练模式的概念分支

优先加载：

1. `platform-basics.md`
2. `response-templates.md`
3. 如果问题最后会落到界面操作，再补 `rich-answering.md`

第一轮输出：

- 先给一句本质定义
- 再告诉用户属于哪一层
- 如果用户下一步要操作，再落到一个最小动作
- 一旦落到具体界面操作，就切到图文增强回答

## I. 周边集成

常见原话：

- `怎么关联 PCS`
- `翻译平台的补丁怎么发到 DCS`
- `应用市场怎么加签`

进入模式：

- 教练模式

优先加载：

1. `release-and-quality.md`
2. `coach-playbooks.md`
3. `response-templates.md`
4. `rich-answering.md`（如果用户是在问具体操作）

## 一句话路由公式

可以用下面这句在脑中先过一遍：

`用户是想学、想办、想排、想让我直接修，还是想让我直接点？`

对应关系：

- 想学 -> 教练模式
- 想办 -> 教练模式快速分支
- 想排 -> 诊断模式
- 想我直接修 -> 修复模式
- 想我直接点 -> 浏览器协作模式

## 重叠意图处理

有些话同时包含“想学”和“想办”，例如：

- `带我一步步创建 DCS 项目`
- `从零开始带我登录并建项目`
- `一步步带我关联环境`

这类请求默认仍进入教练模式，但如果任务本身可通过浏览器完成，首轮必须同时给出浏览器代操作入口。

处理规则：

1. 不要直接切到浏览器协作模式，因为还没有拿到授权。
2. 先按教练模式给当前阶段的第一个动作。
3. 再补一行明确选项：
   - `如果你想我直接打开浏览器帮你做，回我：帮我打开浏览器<当前任务>`
4. 只有用户明确回复授权后，才真正进入浏览器协作模式。

## 证据不足时只问一个问题

优先问能最大化分流效果的问题：

- `你现在最接近哪一步？`
- `这是构建失败、发布失败，还是部署后效果不对？`
- `你现在手上有截图、日志，还是代码仓库？`

不要同时问三四个问题。
