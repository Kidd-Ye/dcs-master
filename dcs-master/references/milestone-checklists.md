# DCS Milestone Checklists

当你需要判断“这一阶段是不是已经完成”“用户现在该不该进入下一步”“最小需要回传什么证据”时，加载本文件。

## 使用方式

每个里程碑都看 4 项：

- 前置条件
- 完成定义
- 最小回传证据
- 不通过时先排什么

只有当前里程碑满足“完成定义”，才推进到下一阶段。

## 里程碑 1：DCS 与 GitLab 账号可用

### 前置条件

- 用户能访问 DCS
- 用户有可用邮箱

### 完成定义

- DCS 登录成功
- 邮箱绑定完成
- GitLab 密码已设置
- 产品侧 `项目管理` 可以完成 GitLab 登录

### 最小回传证据

- 一句状态：`GitLab 已能登录`

### 不通过时先排什么

- 是否收到了设置密码邮件
- 是否使用了绑定邮箱而不是别的邮箱
- 是否需要 Git 密码重置

## 里程碑 2：开发商标识可用

### 前置条件

- 用户知道自己是定制化项目还是 ISV 项目

### 完成定义

- 创建项目时能选出正确开发商标识
- 当前操作人拥有该开发商标识使用权限

### 最小回传证据

- 一张创建项目时开发商标识可选的截图

### 不通过时先排什么

- 组织类型和项目类型是否匹配
- 当前账号是否已加入开发商团队
- 是不是选错项目类型

## 里程碑 3：DCS 项目建好

### 前置条件

- 账号和开发商标识已可用

### 完成定义

- DCS 中存在项目
- 仓库已自动创建
- 至少项目管理员可见项目成员列表和仓库入口

### 最小回传证据

- 项目列表里能看到新项目

### 不通过时先排什么

- 保存项目时是否成功创建 Git 仓库
- 依赖产品、版本、项目类型是否填错

## 里程碑 4：成员体系就绪

注意这里分三种成员能力，不可混淆：

- DCS 项目成员
- 开发商标识团队成员
- 推送权限 / 测试报告审核权限

### 完成定义

- 需要参与开发的人已进入 DCS 项目
- 需要用开发商标识的人已进入开发商团队
- 需要发布或审核的人已被授予对应权限

### 最小回传证据

- 一句状态：`成员已加入项目并具备对应权限`

### 不通过时先排什么

- 只加了 DCS 项目成员，没加开发商团队
- 忘了给推送权限
- 忘了给测试报告审核权限

## 里程碑 5：数据中心关联成功

### 适用场景

- 用户要从产品侧推元数据到 DCS 项目仓库

### 完成定义

- 产品侧项目状态为 `已关联`

### 最小回传证据

- 一张状态为 `已关联` 的截图

### 不通过时先排什么

- 当前 Git 用户是否在 DCS 项目成员中
- 项目是否满足展示条件
- DCS 项目管理员是否已经确认关联请求

## 里程碑 6：环境关联成功

### 适用场景

- 用户要发布到天梯或私有云

### 完成定义

- DCS 环境关联状态为 `已关联`

### 最小回传证据

- 一句状态：`环境关联已成功`

### 不通过时先排什么

- 是否录入了正确租户域名或私有云环境
- 天梯/目标环境是否完成安全码确认
- 是否只是还没刷新同步结果

## 里程碑 7：仓库内容可构建

### 完成定义

- 本次要构建的内容已经真实进入仓库
- 元数据构建至少具备正确 `datamodel` 结构
- 插件构建所需 Gradle 文件、模块和代码已提交
- 静态资源构建时仓库里至少有一个有效元数据

### 最小回传证据

- 一个仓库路径或截图，证明对应内容已存在

### 不通过时先排什么

- 页面有没有连同应用一起推
- `datamodel` 是否为空或缺文件
- Gradle 相关文件是否漏提交
- 静态资源目录是否按要求放在 `webapp/isv/<开发商标识>/`

## 里程碑 8：构建成功

### 完成定义

- DCS 或产品侧构建结果为成功
- 产出了对应制品

### 最小回传证据

- 构建结果状态，或构建记录截图

### 不通过时先排什么

- 构建类型是否选错
- 构建依赖版本是否选错
- 是否命中了常见构建异常：
  - `gradle.properties不存在`
  - `依赖错误`
  - `datamodel内容为空`
  - 类名与文件名不一致

## 里程碑 9：质量与测试报告通过

### 完成定义

- 如果包含代码：质量扫描通过
- 如果要求测试报告：测试报告已提交且审核通过

### 最小回传证据

- 一句状态：`质量通过` 或 `测试报告通过`

### 不通过时先排什么

- 是 Fortify 还是 SonarQube
- 是否只盯着低优先级问题，忽略了高危/严重
- PCS 是否已关联，导致需求清单导入受阻

## 里程碑 10：发布成功

### 完成定义

- 制品已从 DCS/产品侧发布到天梯或私有云平台

### 最小回传证据

- 制品已出现在目标发布列表中

### 不通过时先排什么

- 环境关联是否成功
- 风险等级是否需要审核
- 发布入口是否选错制品

## 里程碑 11：部署验证完成

### 完成定义

- 沙箱或目标环境部署成功
- 用户已完成业务验证
- 如需要，已从沙箱申请到生产

### 最小回传证据

- 一句状态：`沙箱验证通过` 或 `生产已部署成功`

### 不通过时先排什么

- 部署日志里是否提示应用/页面缺失
- 补丁包里是否缺少应用编码 XML
- 是否是应用编码或页面编码冲突

## 漏洞修复专项验收

### 完成定义

- 已定位到具体规则、文件、行号
- 已修改真实仓库或给出准确修复点
- 已重新发起构建/重扫

### 最小回传证据

- 一条具体规则名和对应结果变化

### 不通过时先排什么

- 是否拿到的只是 PDF，总结不够精确
- 是否还没定位到具体文件和行号
- 是否修了非阻塞问题，没修当前卡住的高优先级问题

## 一句话判断是否该继续推进

如果你无法回答下面这句，就不要推进下一步：

`用户当前阶段的完成定义，是否已经被一个最小证据证明了？`
