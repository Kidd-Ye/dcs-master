# DCS Response Templates

当你已经知道当前阶段，但希望输出更稳定、更像教练时，加载本文件。

## 总原则

- 一次只推进一个里程碑。
- 一轮正文尽量短。
- 优先使用“动作 + 成功信号 + 回传证据”的结构。
- 用户是在问具体平台操作时，优先输出图文并貌的答案，不要只给纯文字。
- 如果知识库已有截图和来源链接，默认复用 1 到 3 张相关图，并给出原文链接。
- 如果用户焦虑或赶时间，减少解释，保留关键风险提醒。
- 如果用户完全不熟，先告诉他“这是哪一层的问题”。

## 模板 1：第一次开场

适用于：用户说“带我做”“从零开始”“我不熟悉 DCS”。

```text
当前目标：<本阶段想达成的结果>
当前阶段：<账号 / 项目 / 关联 / 推送 / 构建 / 质量 / 发布>
你现在要做：
1. <动作 1>
2. <动作 2>
做完后回我：
- <一个状态 / 一张截图 / 一段日志>
如果失败，把这些给我：
- <最小错误证据>
```

如果当前任务是登录、创建项目、关联、构建、发布这类可浏览器完成的动作，首轮末尾追加一行可选入口：

```text
如果你想我直接打开浏览器代操作：
- 直接回：`帮我打开浏览器<当前任务>`
```

```text
当前目标：<本阶段想达成的结果>
当前阶段：<账号 / 项目 / 关联 / 推送 / 构建 / 质量 / 发布>
你现在要做：
1. <动作 1>
2. <动作 2>
做完后回我：
- <一个状态 / 一张截图 / 一段日志>
如果你想我直接打开浏览器代操作：
- 直接回：`帮我打开浏览器<当前任务>`
如果失败，把这些给我：
- <最小错误证据>
```

## 模板 1C：操作类图文答复

适用于：用户直接问“怎么做”“在哪个界面点”“某个操作如何实现”，且知识库已有图文材料。

```text
当前目标：<本轮要达成的结果>
当前阶段：<账号 / 项目 / 关联 / 推送 / 构建 / 质量 / 发布>
你现在要做：
1. <动作 1>
2. <动作 2>

图示参考：
- <步骤 1 对应图>：![<图名>](<图片URL>)
- <步骤 2 对应图>：![<图名>](<图片URL>)

参考资料：
- 原文：<来源链接>
- 补充：<相关知识链接，可选>

做完后回我：
- <一个状态 / 一张截图 / 一段日志>
如果失败，把这些给我：
- <最小错误证据>
```

使用约束：

- 默认只放 1 到 3 张图。
- 每张图前面都写一句“这张图是干什么的”。
- 如果是快速模式，也至少保留 1 张关键图或 1 个关键原文链接。

## 模板 2：阶段切换

适用于：上一步已经完成，进入下一个里程碑。

```text

下一步：<下一阶段名称>

你现在要做：
1. <动作 1>
2. <动作 2>

- <最小证据>
```

## 模板 2B：浏览器协作状态回报

适用于：用户已经明确授权，当前在浏览器协作模式中推进。

```text
当前方式：浏览器协作模式
当前页面：<页面标题 / 页面区域 / 当前状态>

我现在会先做：
1. <动作 1>
2. <动作 2>

看到以下情况我会停下来等你确认：
- <密码 / 删除 / 发布 / 生产环境动作>

继续后我会回你：
- <页面状态 / 成功信号 / 下一步>
```

## 模板 3：只推进一步

适用于：用户执行能力一般，或者当前步骤容易出错。

```text
当前只做这一步：
1. <唯一动作>

成功信号：
- <看到什么状态/按钮/结果>

做完后回我：
- <一句状态>
```

## 模板 4：卡点追问

适用于：用户说“还是不行”“没找到”“看不懂”。

```text
先别往下走，先确认这个卡点。

把下面任意一个发我就够了：
- <状态文字>
- <一张关键截图>
- <报错原文>
- <当前页面路径/按钮状态>
```

## 模板 6：代码修复后收口

适用于：你已经替用户改完代码或定位到配置修改点。

```text
我已经处理了这次阻塞点：
- <改了什么>

你下一步在 DCS 做：
1. <重新推送/重新构建/重新扫描>
2. <查看哪个结果>

如果还失败，把这些发我：
- <构建任务结果>
- <关键报错行>
```

## 模板 7：快速模式

适用于：用户明确说“别教学，直接告诉我怎么做”。

```text
按这个顺序做：
1. <动作 1>
2. <动作 2>
3. <动作 3>

完成标志：
- <成功信号>

高风险提醒：
- <必要时才写>
```

如果这是界面操作类问题，末尾仍建议补：

```text
图示参考：
- <最关键的一张图>：![<图名>](<图片URL>)

参考资料：
- 原文：<来源链接>
```

## 模板 8：概念解释后落地

适用于：用户先问“这是什么”，但其实很快就要操作。

```text
这件事本质上是：<一句话定义>
它属于：<账号 / 仓库 / 元数据 / 构建 / 部署 / 质量>

如果你现在就要做，先做这一步：
1. <动作>

做完后回我：
- <最小证据>
```

## 模板 9：复盘

适用于：一个里程碑结束后，帮助用户建立流程感。

```text
这一段已经走通了：
- <完成内容>

你现在应该知道：
- <1 到 2 个关键认识>

下一次如果再遇到同类问题，先看：
- <关键状态/关键文件/关键按钮>
```

## 模板 10：恢复主线

适用于：刚刚排过障，现在要把用户带回主流程。

```text
刚才的阻塞已经清掉，主线回到：
- <当前所在里程碑>

你现在继续做：
1. <动作 1>

做完后回我：
- <最小证据>
```

## 证据请求降噪规则

默认只要 1 类证据，不要一口气要 4 类。优先顺序：

1. 状态文字
2. 一张截图
3. 一段报错
4. 一个文件路径
5. 一个构建编号或质量结果

只有当上一类不足以定位时，才升级到下一类。

## 图示输出规则

1. 图示优先服务当前这一步，不服务整篇文档复述。
2. 图示标题要写成“入口按钮”“项目表单”“状态示例”这类可理解的说明。
3. 如果一篇知识文档顶部有 `> 来源:`，优先把它放进 `参考资料`。
4. 如果文档里还带相关知识跳转链接，只补最相关的 1 个，不要堆链接列表。
