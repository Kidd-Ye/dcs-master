* * 3. Scope of application of thermal deployment * *

(1). Hot deployment only supports public cloud standard tenants, and single tenants do not support hot deployment;

(2). Thermal deployment currently only supports the sandbox environment. The production environment does not support it for the time being. Whether it will be supported in the future depends on the situation.

(3) Single sign-on and third-party login-free plug-ins require regular deployment to take effect.

* * 5. Collaborative development cloud code detection * *

When using the collaborative development cloud for online construction, the collaborative cloud's quality monitoring platform will perform quality testing on the second open code, and at the same time, it will also detect the hot deployment code specification. Only the product package that meets the hot deployment development specification has the permission to [start hot deployment]. The flowchart is as follows:
![image](https://vip.kingdee.com/download/0109aa194a15be9d4f9782ddc4e1b2ae7070.png)
If you want to know whether the two-open code meets the thermal deployment specifications, you can check the [whether hot deployment is supported] column in the [Product List] after the product package is successfully built.If the two-open code conforms to the hot deployment specification, [Yes] will be displayed, otherwise [No] will be displayed.

Note that only the plug-in code needs to judge whether hot deployment is supported. For the package with only metadata, that is, only dm has no jar, so there is no need to pay attention to whether hot deployment is supported.
