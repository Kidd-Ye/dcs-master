Instructions for filling in the fields:

1) Sky template: Kingdee AI Starry Sky recommends the use of "one cloud, one project" engineering template, Star Han project proposal to choose "one application, one project", "one application, multiple projects" engineering template;

2) Developer logo, project identification: automatically brought out from the remote code warehouse project configuration file, users do not need to fill in, can not be modified;

3) Cloud identification: user-defined, fill in the cloud code, required;

4) Application identification: user-defined, fill in the application code, "one cloud, one project" project template is not required, other project templates are required;

5) Server information: According to the actual environment, fill in the resource download address of the development server (the dependency package built by the local project needs to be downloaded from the development server), MC address, zookeeper address (the link needs to be accompanied by the zk account and password);

Step 5: After clicking [Finish], idea will automatically start the creation and configuration of the local project, mainly completing the following operations (no user intervention is required during the whole process, the following 3 steps are all completed automatically by the tool):

1) Create the Gradle project catalog structure according to the project template selected by the user.

2) Automatically download the build-dependent jar packages required for the local project from the user-specified development server.

3) Automatically import the downloaded Sky Project build dependency package and refresh the Gradle Project.
