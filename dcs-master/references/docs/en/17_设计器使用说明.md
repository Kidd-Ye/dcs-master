1 Functional introduction

The Sky Platform Designer is located in the "Development Platform". After the page is created, it automatically enters the designer interface to provide developers with page modification and element configuration functions.The designer interface is divided into 5 blocks: design view, function buttons, control bar, design area, and property bar.

In addition, the form itself has the following functions: name, encoding, page type, source page, modification time, favorite, delete, inherit, move, export, preview, view, etc.
![image](https://vip.kingdee.com/download/010059ad3402f3fa41f593aa322b68ae7afa.png)
1.1 Design View

Taking the document page as an example, its design view is divided into form design, list design, mobile form design, mobile list design, and XML design.

- The form design view interface can be used to design the page layout on the desktop, add various controls and set various properties;

- The list design view interface can be used to design the desktop list interface of the document, * * see [list property description] (https://vip.kingdee.com/article/272105462470920192) * * for details;

- The mobile form design view interface can be used to design the mobile layout interface of the document, * * see * * [* * Mobile page development * *;] (https://vip.kingdee.com/article/272106314451503104)

- The mobile list design view interface can be used to design the mobile list interface of the document, * * see * * [* * Mobile page development * *] (https://vip.kingdee.com/article/272106314451503104);

- * * * The XML Design View interface shows the XML design of all the elements of the page, divided into two types: page metadata and entity metadata. * * *
* Note: Pages are different and have different design views.For example, dynamic forms do not have lists, mobile forms, and mobile lists.Check out the descriptions for each page type. *

1.2 Function buttons

Feature buttons include Save and Preview.

- Save: save the xml metadata generated by the design to the database;

- Preview: Instantly generate a runtime page to verify the design.

 

1.3 Control Bar

* * * Inside the designer * * *:

- Control toolbox: The control toolbox contains common field controls and business field controls accumulated by Kingdee over many years of experience, which are divided into: common fields, business fields, document bodies, common controls, container controls, navigation controls, document lists, chart controls, filter controls, workflows, etc.Different pages support different elements, please check the description of each page type;

- Page Element Outline: The page element outline shows all existing elements of the page in a parent-child hierarchy. Click to switch the focus element.Dragging is supported to adjust the element hierarchy.In addition, the breadcrumbs at the bottom of the design area can partially replace the viewing role of the page element hierarchy;

- The entity element outline hierarchically displays all the entity fields on the page.The Show Only field does not show the control at this time.

* Note: In Sky page development, business fields should be prioritized, and general fields and controls should be selected if no suitable one is available. *
![image](https://vip.kingdee.com/download/01005eeac827a77443b694c067a3aa691fc8.png)
1.4 Design area

The design area is mainly used for form field addition and effect viewing. Features:

- Graphic design;

- What you see is what you get.

The design area consists of three sections:

- Top design buttons to control common page design operations: repeat, delete, move forward, move backward, undo, redo, cut, copy, paste, format brush;

- Central content area, field drag-in addition for documents, form display design, etc.;

- Bottom navigation bar (breadcrumbs) showing all parent controls of the current focus element;

The display of the design area is affected by the layout.

 

1.5 Property Bar

Display the attributes of the focus element, divided into * * business attributes, style attributes, and layout * * 3 categories.Each property item is based on business semantics, and modifying the property will change the content and display effect of the document.Different elements have different properties, please check the description of the corresponding control or page.
