Problem description

After the auxiliary information field is added to the form opened by the customer, the deployment production is found, and the data under the corresponding category is empty and cannot be used;
![image](https://vip.kingdee.com/download/010954e1d77c1a534e229eee65c09087fcca.png)
The reason is that the auxiliary material classification ID of the sandbox environment binding is inconsistent with the production environment auxiliary material classification ID.
![image](https://vip.kingdee.com/download/0109806da493aec4428ca056ce1d51d3c6e8.png)
![image](https://vip.kingdee.com/download/010939424e06b1a9423bbe0f35e3bdd1fc75.png)
Solution:

1. Standard solution reference: [How can the data not be solved by the production environment auxiliary materials?] (https://vip.kingdee.com/article/660530903780510208?productLineId=40
![image](https://vip.kingdee.com/download/0109cd79923c46be4dca8b63b9a027a7d59e.png)
![image](https://vip.kingdee.com/download/01092f9c636079834b9ebdd837507a8e1e90.png)
Note: In the parameter setting, the test environment can be selected as sit or UAT environment, the production environment is selected as production environment, and the link key setting is set according to [link key setting] (https://vip.kingdee.com/article/614091392406412800)

2. Select auxiliary data classification (or other data, such as auxiliary data, set templates, conversion rules, etc.) for cross-environment transmission. Note that it is best not to have the same coded auxiliary data classification in the production environment. If it is manually established in the production environment, it should be deleted.
![image](https://vip.kingdee.com/download/0109577b67493d2a4db3a6d7b47fe4eec9fc.png)
![image](https://vip.kingdee.com/download/0109a6c42259a24c48c6a94519bc9d8bdb7f.png)
3. Select the synchronization of transmission packets in the target environment, and the synchronization status of the synchronization success will be displayed.If it fails, go to the details to see what the reason is, and solve according to the reason.
![image](https://vip.kingdee.com/download/010975bfb2590e624f089d87585616d778ae.png)
Note: The business unit number of the sandbox environment must be modified to match the production environment before being added to the transport package.Otherwise, an error will be prompted when synchronizing.The sandbox environment shown below is generally behind the business unit.test and needs to be removed.Change the number to be consistent with the production environment, otherwise all data created based on the organization will be reported incorrectly.
![image](https://vip.kingdee.com/download/0109aeadd88c5e664167ab493cc23855876b.png)
