Enter the project management - project list interface, click the "New" button in the upper left corner, and jump to the new project interface.

(Tip: A customer only needs to create one project initially, and the development content in the project supports multiple builds and deployments.There is no need to recreate a new project every time you build or deploy.)

II. Fill in the project information
![image](https://vip.kingdee.com/download/010980e3cbb08ee74bb29fab4824cf70ac93.png)
The following information is required for the new project:

[Project name]: User-defined, the customer company name can be used for project naming.

[Project type]: Select the item, please select according to the type:

1) Customer customized development: customer project delivery, which may be delivered by a branch (Kingdee Branch) or a delivery partner. After the code is scanned, the product can be directly deployed on the ladder;

2) ISV standard product development: the development of products by ISV manufacturers in the industry. After the code scanning is passed, the products are finally put on the application market for sale;
![image](https://vip.kingdee.com/download/01099ac095dcd0e2485ebedde2e2ea5589ff.png)
![image](https://vip.kingdee.com/download/0109aa16a5b5c8a3497587912d089f72f520.png)
Note: When selecting the project type, the developer logo will be automatically brought out for users to choose. If the developer logo is not selected, please check the knowledge [How to deal with the developer logo can not be found] (https://vip.kingdee.com/article/536230847855313408?specialId=512315349249697024
![image](https://vip.kingdee.com/download/01090f227422eefa4ccfbd73f713bc5e3b25.png)
[Dependent product version]: Select the product version number that is consistent with the customer's production environment. When building online, the corresponding version of the jar package will be used for reference construction. After the subsequent upgrade of the customer's production environment, the corresponding product version of the project can be modified and updated.
![image](https://vip.kingdee.com/download/0109d67a5728537849cba0a5ab71fd7b3d3d.png)
[List of project members]
![image](https://vip.kingdee.com/download/01096374bfb22de64e8abcf15d8bbbbc7ebf.png)
- Project roles: divided into administrators and developers. The project creator is the project administrator by default, and can perform project management operations, including associating the ladder environment and pushing patch packages. Other members can only develop the project, and the administrator can transfer it.

- Git user role, the creator of the project defaults to the maintainer of the codebase.

- Push patch permissions: set by the project administrator, only members with push permissions can push patches to the ladder tenant environment.

The project is created for the first time, this step cannot be added to other members.For other members to join, you can invite other members to join through the invitation link after the project is successfully saved.

After clicking "Save" in the upper left corner, the pop-up box prompts the user to create a Gitlab repository and authorize project members.Ask the user to click OK.Once determined, the platform will create a repository to hold all the code and metadata for the project, and give the administrator administrative rights to the repository.
![image](https://vip-admin.kingdee.com/download/01094745fea8381543cfbdf133191961a30b.png)
![image](https://vip.kingdee.com/download/01098b4d41f4bc15485f85ca91a7773f093b.png)
[Warehouse List]

1) No need to fill in, the system will automatically assign a Git repository when saving the project.

2) In the repository automatically assigned by the system, the code folder and the datamodel folder are automatically created. The code folder is used to store the plug-in code, and the datamodel folder is used to store the extended metadata.After accessing through the browser by copying the link of the repository, and logging in through the Git mailbox and password, you can see that the directory structure of the repository is as follows.
![image](https://vip.kingdee.com/download/01098c6123fce24f4f3bb1ea9b19b46fce58.png)
The project will be created successfully in a few steps above.Next, we can invite other team members by “inviting members”:

[Member Invitation]

1) Click Invite Members above the list of project members
![image](https://vip.kingdee.com/download/0109ee1e80c3b33949a5a9c778674914719d.png)
2) Copy the invitation link and send it to the member who needs to join.
![image](https://vip.kingdee.com/download/010996ccde88c55a4f59b72cb6163b6e4c5f.png)
3) The invitee can click the link to open the information filling interface, confirm the email address, and apply to join:
![image](https://vip.kingdee.com/download/01095ed09addf4c84996a92541db046e35ae.png)
4) When the invitee applies to join, the administrator clicks "Approved" in the list of project members to join the project.
