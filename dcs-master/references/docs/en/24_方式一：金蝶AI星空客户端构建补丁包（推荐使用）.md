1 Login Product

Log in to the Kingdee AI Sky Client and enter the [Collaborative Development Platform] [Project Management] interface.

2 Metadata construction

There are two ways to build metadata, one is full build, and the other is partial build.

(1) Full Metadata Build: Package all metadata from the GitLab repository in one patch.
![image](https://vip.kingdee.com/download/01090f7a0f03d06c4e33a27a11a87d284b0c.png)
Click "Build to Full" and the pop-up box prompts whether to confirm all metadata in the packaged warehouse.Users need to make sure that the metadata in the repository is up to date, and once deployed to the target environment, the historical version will be overwritten.
![image](https://vip.kingdee.com/download/01091deef954b939475d9443bb81a32c9a38.png)
(2) Partial metadata build: Select from the data center that needs to be pushed to the GitLab repository and initiate the build of the patch package.
![image](https://vip.kingdee.com/download/0109ce69ecfcf1054473ba84e005070f77d2.png)
Enter the "Metadata Push" interface, select the metadata you want to push to the warehouse, and click "Push Warehouse". Once the GitLab repository is pushed successfully, you can build and package the metadata for the pushed success. After the build is successful, the next step is to click "Publish" to push the patch package to the ladder or to the private cloud deployment platform.
