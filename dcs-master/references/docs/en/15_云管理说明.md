1 Functional introduction

In the Sky platform, the cloud can be understood as a division of enterprise business areas, and a cloud is a solution for a business area.The Sky platform provides a variety of functional operations on the cloud, including creation, modification, deletion, and git synchronization operations.This article introduces the application scenarios and main operations of the Sky Business Cloud, and explains the page elements and related terms involved.

2 Scenarios

"Cloud" is one of the most core business designs in the sky, and it is also the business design of the top floor of the sky platform.In the sky, a “cloud” represents a complete business domain solution (e.g., finance cloud, procurement cloud).Typically, a business cloud can handle the business needs of different areas by including multiple applications.In the process of enterprise system delivery, it is usually to select certain applications under one cloud for batch online delivery.

Operation 2: Modify the business cloud

When managing the business cloud, you can change the data of the cloud by clicking on Modify.Among them, the "basic application" has the following instructions:

- Basic applications can be bound to any application under the current cloud. This application is used as the basic application of the current cloud. When the business cloud is checked in, the business cloud will be checked into the svn path of the basic application;

- Basic apps play an important role in app import.The import of apps depends on the integrity of the path. When certain clouds do not exist in the environment, importing apps directly under these clouds will fail because the path cannot be retrieved.However, when the application is the basic application, the system will automatically create the corresponding business cloud based on the basic application information, and then import the application to the cloud.

Operation 3: git operation/svn move-in

Developers can set the synchronization method of the code in the system service cloud, and can use git to submit push or svn to move in.

Switch address: [Configuration tool]→ [System parameters]→ Select [Development platform] under the development service cloud→ to change the version management to git/svn.

* Note: git management and svn management have a complete feature catalog at the application management level.At the cloud management level, only git push and svn move in, mainly for bulk synchronization. *

- When using svn to move in, you need to configure svn management first, and then click the svn move in button at the bottom right of the cloud card to complete the business cloud move in;

- When using the git operation, click the git flag in the bottom right corner of the business cloud card to make git submissions and git push.

Submit: (only to the local repository), the submitted file is verified as the configured local repository address directory.

Push: (Push to remote repository), the pushed file is verified as the configured remote repository address.After the push is completed, there is a push modification comparison for reference.
