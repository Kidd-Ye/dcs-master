# DCS 协同开发云项目看板使用指南

> 分类: 最新功能

## 一、前言

### 1. 看板入口
协同开发云 → 看板 → 项目看板

### 2. 核心价值
项目看板是**构建流程 + 项目质量**一体化数据可视化监控工具，帮您直观掌握项目从构建到部署的全链路状态、核心指标与质量风险，无需手动汇总数据，快速定位问题、优化效率。

## 二、构建看板（全流程监控）

### 1. 核心指标区（一目了然看关键）

![image](https://vip.kingdee.com/download/0100fb8cdb4201e343cbb9bfa2e0f75329a5.png)

| **指标名称**

 | **定义说明**

 |
| 构建成功率

 | 成功构建的任务数 ÷ 总构建任务数（核心效率指标，反映构建稳定性）

 |
| 平均构建时长

 | 所有构建任务的平均耗时（单位：秒，优化方向：缩短构建周期）

 |
| 推送私有云次数

 | 构建完成后，制品推送到私有云环境的总次数

 |
| 推送公有云次数

 | 构建完成后，制品推送到公有云环境的总次数（ISV 产品需关注）

 |
| 提单次数

 | 制品推送后，发起部署提单的总次数

 |
| 提单部署率

 | 部署成功的次数 ÷ 提单总次数（反映提单到部署的转化效率）

 |
| 部署成功率

 | 部署成功（生产 / 沙箱均算成功）的次数 ÷ 总部署次数（核心交付质量指标）

 |

> 注：日期范围支持自定义，默认展示最近 7 天数据，趋势图自动抓取截止时间前 10 天的历史数据。
> 
> 

### 2. 分布图解读（快速定位关键维度）

![image](https://vip.kingdee.com/download/01004ff74201156c40b7abcafc4dbc126c24.png)

| **图表名称**

 | **核心作用**

 | **解读要点**

 |
| 构建内容分布

 | 查看不同构建类型的占比

 | 支持 “插件代码”“全量元数据 + 插件代码” 等类型；选择 “插件代码” 构建需确保项目 code 目录有 Java 源文件

 |
| 构建错误分布

 | 分析构建失败的主要原因

 | 按失败类型占比排序，快速锁定高频问题（如依赖缺失、代码语法错误）

 |
| 构建来源分布

 | 统计构建请求的发起渠道

 | 如 “DCS”“标品” 等，了解团队构建场景集中地

 |

### 3. 趋势图 & 柱状图（追踪变化趋势）

![image](https://vip.kingdee.com/download/01008782c84a66ed410cb86de17d41b7db92.png)

| **图表名称**

 | **核心作用**

 | **解读要点**

 |
| 构建次数 & 成功次数趋势

 | 跟踪每日构建总量与成功量的变化

 | 若成功次数持续低于构建次数，需排查共性构建问题

 |
| 构建次数 & 推送次数趋势

 | 关联构建与制品推送的衔接效率

 | ISV 产品需关注公有云 / 私有云推送的同步性

 |
| 平均构建时间趋势

 | 监控构建耗时的波动情况

 | 若耗时突然飙升，需检查构建配置、依赖加载等环节

 |
| 构建 / 提单次数统计

 | 对比每日构建量与提单量的匹配度

 | 若构建多、提单少，可能存在制品推送或部署需求衔接问题

 |

### 4. 明细 & 统计表（精准追溯详情）

#### （1）构建明细信息表（近 10 条）

![image](https://vip.kingdee.com/download/010053d747d353d54b1b849b884d0dc608b9.png)

- 展示字段：构建时间、构建任务 ID、构建内容、构建结果、构建耗时、部署状态、质量报告状态、制品来源、制品包名称、制品版本号、构建依赖版本

- 核心用途：快速追溯单个构建任务的全量信息，定位失败任务的具体原因（如依赖版本不兼容、构建内容有误）

#### （2）开发者构建推送统计表

![image](https://vip.kingdee.com/download/01000350b3ff810941c8bdde4dfd2a9ef3b7.png)

- 展示字段：开发者姓名、构建次数、推送次数

- 核心用途：按开发者维度评估工作效率，识别高频构建 / 推送的核心人员，优化团队协作分配

## 三、质量看板（风险精准把控）

### 1. 核心指标区（质量状况一眼掌握）

![image](https://vip.kingdee.com/download/01005362204cf33b4d918d2590f55b795360.png)

| **指标名称**

 | **定义说明**

 |
| 质量漏洞总数

 | 所有风险等级的质量问题总数量

 |
| 高危问题总数

 | 高风险质量问题数量（需优先修复，避免影响系统稳定性）

 |
| 中危问题总数

 | 中等风险质量问题数量（建议迭代修复）

 |
| 低危问题总数

 | 低风险质量问题数量（可按需优化）

 |
| 严重问题总数

 | 致命风险质量问题数量（阻断性问题，需立即修复）

 |
| 缺陷文件总数

 | 包含质量问题的文件总数量（反映问题覆盖范围）

 |
| 平均检测时长

 | 质量扫描的平均耗时（单位：秒，优化扫描效率）

 |
| 检测次数

 | 质量扫描的总执行次数

 |
| 各等级已忽略数

 | 手动标记为 “忽略” 的各风险等级问题数量（需合理管控忽略权限）

 |

### 2. 分布图解读（质量问题深度分析）

![image](https://vip.kingdee.com/download/010026776fe6e38b4decaa70b889c8b3b233.png)

| **图表名称**

 | **核心作用**

 | **解读要点**

 |
| 检测等级分布

 | 查看不同风险等级问题的占比

 | 若高危问题占比超 30%，需重点整改代码质量

 |
| 开发内容分类

 | 统计质量问题覆盖的开发内容类型（如代码、元数据、SQL）

 | 若 “代码” 类问题占比高，需强化编码规范培训

 |
| 检测类型分布

 | 分析问题所属检测维度（规范、安全、性能）

 | 安全类问题需优先修复，避免数据泄露或系统漏洞

 |
| 质量扫描引擎分布

 | 查看各扫描工具的使用占比（如 eslint）

 | 了解质量检测的工具覆盖情况，确保检测全面性

 |

### 3. 趋势图 & 条形图（跟踪质量变化）

![image](https://vip.kingdee.com/download/0100731042bf4000471e93df568210367661.png)

| **图表名称**

 | **核心作用**

 | **解读要点**

 |
| 开发项目质量趋势图

 | 跟踪每日缺陷数量的变化

 | 若缺陷数量持续上升，需排查编码规范执行或扫描规则配置问题

 |
| TOP10 质量问题

 | 展示出现频次最高的前 10 类质量问题

 | 按问题频次排序，优先优化高频问题（如 “找不到变量 / 函数”“未使用赋值需删除”）

 |

### 4. 开发者问题统计表

![image](https://vip.kingdee.com/download/010049194a7eb691416ca9d01fd18e3c6965.png)

- 展示字段：开发者姓名、严重问题数、高危问题数、中危问题数、低危问题数、问题总数、开发者资质（是否通过精一认证）

- 核心用途：按开发者维度评估代码质量，针对性开展培训；结合资质认证状态，优化团队质量管控策略

## 四、使用技巧（高效用看板）

- 日期筛选：按需调整时间范围，对比不同周期的指标变化（如周度 / 月度构建��率提升情况）。

- 问题追溯：从 “TOP10 质量问题” 定位高频缺陷 → 查看 “开发者问题统计” 找到对应负责人 → 结合 “构建明细” 追溯关联构建任务。

- 效率优化：若 “平均构建时长” 过长，查看 “构建内容分布” 是否存在全量构建占比过高，可优化为增量构建。
