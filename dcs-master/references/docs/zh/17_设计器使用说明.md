1 功能介绍

苍穹平台设计器位于“开发平台”中，在页面创建完成后，自动进入设计器界面，为开发者提供页面修改和元素配置功能。设计器界面内分为5大块：设计视图、功能按钮、控件栏、设计区域、属性栏。

另外，表单本身还有如下功能操作：名称，编码，页面类型，源页面，修改时间、收藏，删除、继承，移动，导出，预览，视图等。

![image](https://vip.kingdee.com/download/010059ad3402f3fa41f593aa322b68ae7afa.png)

 

1.1 设计视图

以单据页面为例，其设计视图分为表单设计、列表设计、移动表单设计、移动列表设计、XML设计。

- 表单设计视图界面可用来设计PC端页面布局，可添加各种控件并设置各项属性；

- 列表设计视图界面可用来设计单据的PC端列表界面，**详见[列表属性说明](https://vip.kingdee.com/article/272105462470920192)**；

- 移动表单设计视图界面可用来设计单据的移动端布局界面，**详见**[**移动页面开发**；](https://vip.kingdee.com/article/272106314451503104)

- 移动列表设计视图界面可用来设计单据的移动端列表界面，**详见**[**移动页面开发**](https://vip.kingdee.com/article/272106314451503104)；

- ***XML设计视图界面展示了页面所有元素的XML设计，分为页面元数据和实体元数据两种。***

*注意：页面不同，其设计视图不同。例如，动态表单没有列表、移动表单和移动列表。具体区别请查看各种页面类型的介绍。*

1.2 功能按钮

功能按钮包括保存和预览。

- 保存：把设计生成的xml元数据保存到数据库；

- 预览：即时生成运行态页面，验证设计效果。

 

1.3 控件栏

***设计器内***：

- 控件工具箱：控件工具箱内包含通用字段控件和经过金蝶多年经验积累的业务字段控件，分为：通用字段、业务字段、单据体、通用控件、容器控件、导航控件、单据列表、图表控件、过滤控件、工作流等。不同的页面支持不同的元素，具体请查看各种页面类型的介绍；

- 页面元素大纲：页面元素大纲以父子层级关系展示页面现有全部元素，点击即可切换焦点元素。支持拖拽以调整元素层级关系。此外，设计区域底部面包屑可部分替代页面元素层级的查看作用；

- 实体元素大纲分层展示页面上的所有实体字段。此时只显示字段不显示控件。

*注意：在苍穹页面开发中，应优先使用业务字段，如果没有合适的再选择通用字段和控件。*

![image](https://vip.kingdee.com/download/01005eeac827a77443b694c067a3aa691fc8.png)

 

1.4 设计区域

设计区域主要用于表单字段添加和实现效果查看。特点：

- 图形化设计；

- 所见即所得。

设计区域包括三个部分：

- 顶部设计按钮，用来控制页面设计常用操作：重复、删除、前移、后移、撤销、重做、剪切、复制、粘贴、格式刷；

- 中部内容区，用于单据的字段拖入添加，表单显示设计等；

- 底部导航栏（面包屑），展示当前焦点元素的所有上级控件；

设计区域的显示效果受布局影响。

 

1.5 属性栏

显示焦点元素的属性，分为**业务属性、样式属性、布局**3大类。各属性项均基于业务语义，对属性的修改会改变单据的内容和显示效果。不同元素拥有不同的属性，具体请查看对应控件或页面的说明。

3 主要操作

**设计器外的主要操作：**

进入设计器之前，开发者可以对表单进行多项主要操作，如下图和表所示。

| **操作名/按钮名** | **说明** |
| 页面过滤 | 位于表单管理界面上部，按照**设备、页面、排序**对下方的页面内容进行过滤 |
| 页面搜索 | 位于表单管理界面的右上角，可以按编码/名称/id模糊搜索页面 |
| 卡片视图
 | 表单管理界面显示为卡片视图（默认） |
| 列表视图 | 表单管理界面显示为列表视图，此时部分按钮调整至列表工具栏位置 |
| 收藏
 | 收藏选中表单 |
| 修改 | 修改选中表单（进入设计器） |
| 删除 | 删除选中表单 |
| 预览与调试
 | 预览表单页面，和设计器内的【预览】效果一致 |
| 页面与继承关系 | 弹出页面，以树状图逐级展开方式查看当前页面的扩展继承关系 |
| git操作
 | 使用git管理代码时出现，包括提交、推送、还原、拉取。请参考[应用管理](https://vip.kingdee.com/article/272105484633622528) |
| svn迁入
 | 使用svn管理代码时出现，用于迁入svn。请参考应用管理 |
| svn管理
 | 使用svn管理代码时出现，包括比较、日志、更新、删除、锁定、解锁和清理。

- 比较：比较表单的本地版本和云端版本，可以推送到svn或更新到本地；

- 日志：查看svn提交记录，可以对比当前选中的版本与svn最新版本比较，并可以更新本地和提交svn操作；可以对应两版本的差异；可以选中某一版本更新到本地；

- 更新：将svn最新版本的内容覆盖本地数据库的内容；

- 删除：删除svn对应的资源文件；

- 锁定：svn锁定的情况下，本人和他人不能svn签入和svn删除，只有解锁的情况下才可以操作；在SVN上还可以查看被谁锁定了

- 清理：清理本地svn工作区

 |
| 布局 | 新建布局，可以选择空白布局，或直接继承当前表单生成布局。一个单据可继承出多个布局，在功能下（如基础资料的列表布局）可以选择其中一个布局展示。 |
| 扩展与继承
 | - 功能说明：继承和扩展均属于扩展开发，是苍穹提供的对系统原生单据的功能补充，决有限开放性问题，降低标准补丁升级所需要考虑的兼容性影响范围；

- **区别：扩展是对源单实体进行改造增强，扩展之后系统中只存在1个实体—扩展之后的实体。 继承是对源单实体继承之后再改造，继承之后系统中存在2个实体**：源单实体和继承实体。

- [](https://vip.kingdee.com/knowledge/specialDetail/218022218066869248?category=218033944568893696&id=234691132842907136)入口：表单的扩展继承入口与所处应用的扩展情况有关。当前应用是原生应用时（应用上的标志为“原”），表单只可选继承功能；当前应用为扩展应用时（应用上的标志为“扩”），页面可选继承和扩展功能
