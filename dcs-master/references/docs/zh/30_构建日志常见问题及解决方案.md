## **1 ** 缺少bos、biz、trd、cus 依赖
    关键字：

    compileJava FAILED 

    error: package kd.bos does not exist

**1.1 方案一：根据构建日志的报错 引入对应的单个依赖**

**    1）**报错关键字：

    error: package kd.bos.bill does not exist   import kd.bos.bill.AbstractBillPlugIn;

    报错如下图

![image](https://vip.kingdee.com/download/0109bd515614e0d5417da8a0b4fc162e2604.png)

**    修改方案**

    找到 kd.bos.bill.AbstractBillPlugIn对应的jar包为 bos-form-core-5.0.jar，在对应模块的build.gradle的dependencies 里面加上引入该jar的配置 compile fileTree(dir: bos, include: 'bos-form-core*.jar')          如下图

![image](https://vip.kingdee.com/download/0109769dae8957b54e1aa393a780664a49f5.png)

**
**

**2）**报错关键字：

    error: package kd.bos.form does not exist    import kd.bos.form.ShowType;

     error: package kd.bos.list does not exist    import kd.bos.list.ListShowParameter;

    报错如下图

![image](https://vip.kingdee.com/download/0109720d820382554eb3a62b69b18e3fe7d9.png)

    **修改方案**

    找到 kd.bos.form.ShowType和kd.bos.list.ListShowParamete 对应的jar包都为 bos-form-metadata-5.0.jar，在对应模块的build.gradle的dependencies 里面加上引入该jar的配置 

compile fileTree(dir: bos, include: 'bos-form-metadata*.jar')      如下图

![image](https://vip.kingdee.com/download/0109ddeee13ba8754c27ae278bdc7562f8b4.png)

**
**

**1.2 方案二****：** 引入所有的依赖（不推荐使用，会影响构建效率）

**    构建日志报错如下图
**

![image](https://vip.kingdee.com/download/0109983e63ce17ea403b962955df6b899479.png)

**修改方案**

**   **对应模块的build.gradle的dependencies 里面 加上引入bos、biz、trd、cus 依赖配置，

    compile fileTree(dir: bos, include: '*.jar')
    compile fileTree(dir: biz, include: '*.jar')
    compile fileTree(dir: trd, include: '*.jar')
    compile fileTree(dir: cus, include: '*.jar')

如下图

![image](https://vip.kingdee.com/download/010900d78045d7654c6faae34255879cbc4a.png)
